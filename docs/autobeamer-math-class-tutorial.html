Making a Beamer Deck for Your Math Class

A hands-on AutoBeamer walkthrough — written by actually building a Mean Value Theorem lecture end-to-end, gotchas and all.

AutoBeamer v1.2 · passive-study mode · XeLaTeX

What this is

AutoBeamer is a set of seven skills that turn an idea into a polished XeLaTeX Beamer deck through a fixed pipeline instead of guesswork. For a math class you mostly drive one skill, autobeamer-create, and it hands off to the others as needed.

SkillWhat it does for you
autobeamer-createThe router. Picks a mode, plans the deck, drafts frames, runs the quality loop.
autobeamer-buildCompiles with XeLaTeX, fixes font / build errors.
autobeamer-layoutTemplate choice, image scaling, column balance, density.
autobeamer-tikzQuality standards for hand-drawn TikZ diagrams.
autobeamer-validateAutomated checks + the mandatory PDF visual-check.
autobeamer-reviewRubric scoring, pedagogy audit, devil's-advocate pass.
autobeamer-doctorEnvironment preflight — checks deps and records the visual-check ceiling.

STEP 0Pick the right mode

Mode is the single most important choice — it changes density, tone, and the acceptance gates. For teaching a class, you almost always want passive-study.

ModeUse whenMath-class fit
passive-studyThe deck itself must teach: define background, motivate, give worked examples, exercises, glossary.✅ Best default for a lecture students study from.
active-socraticGuided discovery: one question per frame, learner attempts before answers.Good for a flipped / problem-solving session.
academic-presentationA live talk, seminar, or defense — telegraphic, time-boxed.Use for a conference talk, not a teaching deck.
In a chat you would just say: “Create a passive-study Beamer deck teaching the Mean Value Theorem for my Calculus I class.” AutoBeamer loads references/modes/passive-study.md and the matching acceptance gates.

STEP 1Know the pipeline (don't skip phases)

# autobeamer-create runs these in order — each feeds the next
Phase 0  MATERIAL ANALYSIS   read source / extract figures   (skip if idea-only)
Phase 1  NEEDS INTERVIEW     audience, duration, mode         (asks only what it can't infer)
Phase 2  STRUCTURE PLAN      section + slide outline          <-- you approve here
Phase 3  DRAFT              write frames in 5-10 slide batches, compile each batch
Phase 4  FIGURES            TikZ / pgfplots / source images
Phase 5  QUALITY LOOP       compile -> validate -> visual-check -> review -> fix

For a class deck the plan in Phase 2 should name, per section, the motivation → formal object → worked example chain. That is the passive-study contract.

STEP 2Build a real deck

Here is the actual preamble and one teaching frame from the MVT deck used to write this guide.

Preamble

% mvt-lecture.tex  — put this file at the repo root (see Gotcha G5)
\documentclass[aspectratio=169,10pt]{beamer}
\usepackage[teal]{template-lib/template-lib}   % theme
\uselayout{text}\uselayout{eq}\uselayout{twocol}  % only what you use
\usetikzlibrary{arrows.meta}
\TLsetfoot{Calculus I \textperiodcentered{} The Mean Value Theorem}

A teaching frame (the passive-study pattern)

\begin{frame}{The Mean Value Theorem}
  \TLinfoblock{Statement}{If $f$ is continuous on $[a,b]$ and
  differentiable on $(a,b)$, then there exists $c\in(a,b)$ with
  \[ f'(c)=\frac{f(b)-f(a)}{b-a}. \]}
  In words: some tangent is parallel to the endpoint secant.
\end{frame}
Hard rules baked into the mode: no overlays (\pause/\only/…), at most 3 colored boxes per slide, a worked example within 2 slides of every definition, and never \tiny. The validator enforces them.

Compile — always twice

# from the repo root
xelatex -interaction=nonstopmode -output-directory=build mvt-lecture.tex
xelatex -interaction=nonstopmode -output-directory=build mvt-lecture.tex
# or: ./build.sh mvt-lecture

The second pass resolves cross-references and equal height group data written to .aux on the first pass. The real run compiled cleanly to an 18-page PDF.

STEP 3Validate, then look with your eyes

3a · Static gates — PASSED

$ python3 tools/validate_deck.py static mvt-lecture.tex --mode passive-study
PASS static validation: mvt-lecture.tex

This confirmed the passive-study gates objectively: glossary/notation present, exercise frame present, references present, \appendix before the backup solution, no overlays, ≤3 boxes/slide. This part is solid — trust it.

3b · Visual-check is mandatory — and here's why

Beamer suppresses overflow warnings inside blocks and TikZ pictures, so a deck can compile “clean” and still be broken. Render the PDF and actually look:

pdftoppm -png -r 150 build/mvt-lecture.pdf _slides_png/slide

3c · Layout audit

python3 tools/check_layout.py mvt-lecture.tex build/mvt-lecture.log --advise

It now understands prose, equations, and \TL* blocks, so it scores math/text decks sensibly (no more false “sparse” or GV-1 — see G2) and reliably catches real overflow (G1). Still treat the absolute U target band as a guide: text decks legitimately run lighter than image-and-card decks.

Gotchas I actually hit (and what to do)

These all came out of really running the workflow on the MVT deck — and they have all since been fixed. The notes below explain what was wrong and how it was fixed, so you understand the tooling.

G1 · The overflow detector was blind FIXED

check_layout.py searched the log for "...pt too large", but XeLaTeX writes "...pt too high" for vertical overflow. So it reported “Overflow frames: 0” for the deck whose TikZ slide overflowed by 92 pt. Fixed to accept both spellings, with a regression test.

Until you’re sure you’re on the fixed build, confirm overflow yourself: grep "Overfull \vbox" build/deck.log.

G2 · Layout audit mis-scored text/math decks FIXED

The U/B/G/DGV model only knew the legacy card macros, so it didn’t recognise any \TL* command. On the math lecture it flagged all 11 frames as GV-1 “loose text” and “sparse” (utilization ≈0.10–0.26) — even the visibly ~70%-full slide above.

Fix: GV-1 was removed (the skills bless plain body text); check_layout now credits prose, bullets, display math, \TL* blocks and TikZ toward utilization, and only calls a frame “sparse” when it is both low-fill and has no substantive element. The same deck now reports 0 grammar violations and only the genuinely-thin backup slide as light.

G3 · Overflow page attribution was off FIXED

After G1 the tool detected the overflow but pinned it to the wrong frame, because section-divider slides shift PDF page numbers away from frame ordinals. Fix: it now counts every slide-emitting construct (title page, \TLsection, frames) to map each frame to its real PDF page.

G4 · A stale reference file FIXED

references/workflows/full-create-guide.md carried an old duplicated skill-frontmatter block (legacy triggers) that could be mis-registered as a second skill. Fix: the frontmatter was stripped so it’s a plain reference doc like its siblings.

G5 · Your deck must “see” the template library DOCUMENTED

\usepackage[teal]{template-lib/template-lib} only resolves if your .tex sits at the repo root. Put it elsewhere and XeLaTeX can’t find the package. This is now written up in the autobeamer-build skill; keep the deck at the root or point TeX at the library:

# keep the deck anywhere by pointing TeX at the library:
TEXINPUTS=".:/path/to/auto-beamer//:" xelatex -output-directory=build deck.tex

G6 · TikZ diagrams overflow silently FIXED IN EXAMPLE

The most common content problem (and the one above): a diagram is taller than the ~70 mm of usable slide height. Beamer won’t error — it just runs off the edge. Fixes, in order: shrink yscale, give the figure its own frame, or add [c] to the frame (\begin{frame}[c]{…}). Compute marker coordinates with \pgfmathsetmacro so dashed lines actually land on the curve.
